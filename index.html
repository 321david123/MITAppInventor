Challenge 2: Camera Component Redesign

The `useFront` property fails on newer Android due to deprecated APIs and shifting permission models post Android 10. My proposal introduces a new method `TakePicture(auto: boolean)` to allow background image capture if permissions allow, and a `preferredCamera: "front" | "back"` property. I would update `Camera.java`, `Form.java`, and integrate changes into the GWT interface. This would use CameraX for compatibility. I can provide the full proposal if needed.
